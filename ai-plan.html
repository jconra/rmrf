RMRF — Making the AI Decide Like a Human

Plan drafted 2026-07-01 · lives at rmrfbase.com/ai-plan.html · from the AI-audit session

The honest starting point: Claude does not currently see all the AI's reasoning. aiEvents is a rolling 80-entry log that only fires on changes (deploys, kills, mission switches) — it's the narrative, not every decision. The real per-tick reasoning (the view a unit sees — threat, line-of-sight, canBear, bail-HP — and the think() output) is not logged at all. Findings this session came from inference over ~1 Hz state samples ("ammo barely moved + HP dropping → ineffective siege"), not from reading the transcript. That blind spot is why fixes came slowly.

Layer 1 — See the real reasoning CHEAP · HIGH VALUE

Add a debug-gated "flight recorder" to the AI. When a unit's decision changes, emit a structured record: the inputs that drove it (threat / LOS / canBear / bailHP / enemyEliminated…) and the exact branch that fired. Then analysis reads why a unit froze instead of guessing.

This alone would have turned the multi-hour "why is the jotun idle at the wall" hunt into a single glance.

Layer 2 — A systematic tuning loop TRACTABLE TODAY

Today: change a rule by hand → run a batch → eyeball it. Make it a closed loop: change a knob → auto-run N matches → agg.cjs scores it → keep if better. The headless sim harness (~/pw/ai-sim.cjs) is already a training / evaluation gym — thousands of fast, faithful matches. With it we can auto-tune the personality + threshold knobs (grid / evolutionary search) against a fitness score: decisive-win-rate, time-to-breach, army preserved. No neural nets required.

Layer 3 — A better decision architecture THE "LIKE A HUMAN" PART

The brain today is a hand-written if-else ladder of tuned thresholds. Its ceiling is however many rules we can enumerate — every gap we found (the frozen jotun, the un-closed match) was a rule we hadn't written. Two structural upgrades close that gap the way judgment does:

  1. Utility scoring instead of the ladder: each candidate action gets a score from weighted considerations (danger, opportunity, self-preservation, objective-progress); pick the max. It degrades gracefully — when no rule "matches," it still picks the least-bad option instead of freezing. Would have prevented the idle-at-the-wall bug structurally.
  2. Goal chaining / planning at the commander level: "to win → flag → HQ down → turrets down → I have a jotun → go kill turret A." A human plans backward from the win; the AI mostly reacts to the instant. This is what the AI-Lab / BrainGraph groundwork was heading toward — a data-driven brain you can restructure without rewriting code.

Layer 4 — Actual learning (RL) LONG HORIZON

The aspirational ~/todo item. The harness is the missing prerequisite (can't train without a fast sim), so it's now possible — but it's a big project (state encoding, reward shaping, training infra). Reach for it only after Layers 1–3, which get most of the benefit for a fraction of the effort.

One honest caveat

"Better decisions" ≠ "optimal." RMRF is built for fun, and the personalities — imperfect aim, distinct doctrines, a Rogue that sneaks vs a Turtle that holds — are a feature, not something to optimize away. Aim the machinery at "no dumb / frozen moments," not "unbeatable." A perfectly efficient AI would probably be less fun than a characterful, occasionally-flawed one.

Recommended next step

Build Layer 1 (flight recorder) + Layer 2 (auto-A/B loop). Together they turn AI work from "Claude guesses and hand-patches" into "the game tells us exactly what it was thinking, and the harness proves whether a change helped." That's the process upgrade that makes everything after it faster — including the eventual utility refactor or RL.

Where things stand (commit 161df0d): four AI fixes shipped — HQ-as-siege-target, the Closer (eliminated enemy → field a Jotun), the Finisher (unopposed sieger closes in instead of idling), and Preservation (a losing commander turtles under tower cover). Data-identified next bottleneck: reliable base breach (attacker grinds turrets to the last 1–2 / HQ, then stalls).
