--- SHIP: GATE-BAND HUG (js/main.js cellBlocked, knob RR.setGateBand / ?nogateband) ---------
The bug (night verdict residual #2): a SHUT gate's collision slab in blockedFor is
`halfNorm + VEH_R` each side of the door plane — WIDER than the nav grid's blocked gate cells.
So A* could legally route a hull-width path along the closed door that the collision layer then
refuses, leaving the omni hull pressing the band and CREEPING ~0.15u/s. Real progress, so the
progress-gated stuck-escalation never fires — but the net<20u alarm does (and the unreachable-
GOTO contract violation piles up every tick the driver refuses the nav goal).

The fix: mirror the slab in cellBlocked, with the same planning margin the obstacle test uses
(halfNorm + VEH_R + 0.9·VEH_R across the plane, halfRun + 0.9·VEH_R along it). Runs AFTER the
corridor early-return (gateCells) so the single-file throat stays open; skipped for flyers and
for any gate that doesn't block THIS unit (gateBlocks per-team) — so it can't strand an ally and
it reopens the instant the gate is breached or held. 20u cull keeps it cheap in the A* hot loop.
